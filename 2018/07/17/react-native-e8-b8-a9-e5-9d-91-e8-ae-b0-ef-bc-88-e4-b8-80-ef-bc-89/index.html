react-native踩坑记（一）

Author: Melor Date: 2018-07-17 Updated On: 2018-09-13
Categories: app开发
Tags: react-native
什么都懂一点，生活更美好一些

由于业务发展，入坑开始写react-native，发现还挺不靠谱的。自备梯子，搭建好环境后，按照官方文档初始化应用，马上就红屏报错 Unable to resolve module `AccessibilityInfo` from xxx 然后官方给出一条迷之提示

This might be related to https://github.com/facebook/react-native/issues/4968

To resolve try the following:

  1. Clear watchman watches: `watchman watch-del-all`.

  2. Delete the `node_modules` folder: `rm -rf node_modules && npm install`.

  3. Reset Metro Bundler cache: `rm -rf $TMPDIR/react-*` or `npm start -- --reset-cache`.  4. Remove haste cache: `rm -rf $TMPDIR/haste-map-react-native-packager-*`.

我用windows的系统怎么就给出linux的命令呢？‘百度’了半天，最后发现应该是版本问题，降级使用react-native@0.55.4就可以解决这个红屏报错的问题，为这种事折腾半天，不值得。。。 解决方案参考

其中那个给出降级的方案测试可行。

—-2018.7.18更新—— 通过工具react-native-git-upgrade也可以优雅解决降级问题，

npm install -g react-native-git-upgrade
react-native-git-upgrade 0.55.4

这样也能快速更改项目中react-native的版本，经过测试，发现的确0.56.0版本是有bug的，上面的问题无论重复多少遍都会复现，所以目前能用的最新版本应该是0.55.4了
