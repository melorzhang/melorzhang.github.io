v-bind 缩写

<!-- 完整语法 -->
<a v-bind:href="url">...</a>

<!-- 缩写 -->
<a :href="url">...</a>

v-on 缩写

<!-- 完整语法 -->
<a v-on:click="doSomething">...</a>

<!-- 缩写 -->
<a @click="doSomething">...</a>

静态数据vs动态数据

静态数据指的是别的数据更新后，这个数据的值不会受影响的数据，静态数据也是能改变的； 动态数据指的是别的数据更新后，这个数据也会随之更新的数据，动态数据也是可以自行改变的，但需要设置setter方法去更新关联的静态数据。 静态数据放在data里，动态数据放在computed里， 一般情况下computed里的[key]都是function的函数名，[value]是function的返回结果；

data: {
    firstName: 'Foo',
    lastName: 'Bar'
},
computed: {
    fullName: function () {
        return this.firstName + ' ' + this.lastName
    }
}

但也可以使用对象设置属性的setter

computed: {
  fullName: {
    // getter
    get: function () {
      return this.firstName + ' ' + this.lastName
    },
    // setter
    set: function (newValue) {
      var names = newValue.split(' ')
      this.firstName = names[0]
      this.lastName = names[names.length - 1]
    }
  }
}

若静态数据发生改变后，需要执行某些操作，需要在watch属性下注册该数据，

this的指向

vue内this的指向需要使用者对this的指向有较为清楚的理解，因为this都需要自己指向到当前的vue实例，不然很容易踩坑。这点对使用者其实不是太友好的。

class的绑定

使用计算属性解决，

<div v-bind:class="classObject"></div>


data: {
  isActive: true,
  error: null
},
computed: {
  classObject: function () {
    return {
      active: this.isActive && !this.error,
      'text-danger': this.error && this.error.type === 'fatal'
    }
  }
}

也可以使用数组

<div v-bind:class="[{ active: isActive }, errorClass]"></div>

这种也可以的兼容，好不好就各有看法啦， 对于组件来说，组件的class会直接挂载到组件渲染后的根节点上，

绑定内联样式

通过对象的方式绑定样式.根据文档说法，支持驼峰和连缀的写法，但是建议使用驼峰写法。

<div v-bind:style="styleObject"></div>


data: {
  styleObject: {
    color: 'red',
    fontSize: '13px'
  }
}

没错，也可以使用数组

<div v-bind:style="[baseStyles, overridingStyles]"></div>

据文档说明，样式前缀vue会自动加载，不用考虑。 从 2.3.0 起你可以为 style 绑定中的属性提供一个包含多个值的数组，常用于提供多个带前缀的值，例如：

<div :style="{ display: ['-webkit-box', '-ms-flexbox', 'flex'] }"></div>

这样写只会渲染数组中最后一个被浏览器支持的值。在本例中，如果浏览器支持不带浏览器前缀的 flexbox，那么就只会渲染 display: flex。

条件渲染

参考例子使用，

<div v-if="type === 'A'">
  A
</div>
<div v-else-if="type === 'B'">
  B
</div>
<div v-else>
  Not A/B
</div>

当包裹元素为多个，使用template包裹使用，template不会编译节点，

<template v-if="ok">
  <h1>Title</h1>
  <p>Paragraph 1</p>
  <p>Paragraph 2</p>
</template>

· 当 v-if 与 v-for 一起使用时，v-for 具有比 v-if 更高的优先级。

防止输入数据保留，使用key属性

下面例子中，如果不使用key属性，在切换后，输入框会保留之前的数据，当然，不使用key，使用v-modul绑定数据也是可以的

<template v-if="loginType === 'username'">
  <label>Username</label>
  <input placeholder="Enter your username" key="username-input">
</template>
<template v-else>
  <label>Email</label>
  <input placeholder="Enter your email address" key="email-input">
</template>

v-show

<h1 v-show="ok">Hello!</h1>

v-show 不支持 <template> 元素，也不支持 v-else。 v-if 是“真正”的条件渲染，因为它会确保在切换过程中条件块内的事件监听器和子组件适当地被销毁和重建。 v-if 也是惰性的：如果在初始渲染时条件为假，则什么也不做——直到条件第一次变为真时，才会开始渲染条件块。 相比之下，v-show 就简单得多——不管初始条件是什么，元素总是会被渲染，并且只是简单地基于 CSS 进行切换。 一般来说，v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。
