组件

本节主要记录组件的使用，也是vue最重要的一个环节，内容较多，可能会分割成两篇来写

使用组件

全局注册

注册一个全局组件，可以使用 Vue.component(tagName, options)。例如：

Vue.component('my-component', {
  // 选项
})

在实例化vue之前必须保证组件已经注册，

<div id="example">
  <my-component></my-component>
</div>


// 注册
Vue.component('my-component', {
  template: '<div>A custom component!</div>'
})

// 创建根实例
new Vue({
  el: '#example'
})

局部注册

与全局注册相对，这样注册的组件只能在注册的作用域内使用，一般使用这种方法注册的情况更普遍

var Child = {
  template: '<div>A custom component!</div>'
}

new Vue({
  // ...
  components: {
    // <my-component> 将只在父组件模板中可用
    'my-component': Child
  }
})

DOM 模板解析注意事项

当使用 DOM 作为模板时 (例如，使用 el 选项来把 Vue 实例挂载到一个已有内容的元素上)，你会受到 HTML 本身的一些限制，因为 Vue 只有在浏览器解析、规范化模板之后才能获取其内容。尤其要注意，像 <ul>、<ol>、<table>、<select> 这样的元素里允许包含的元素有限制，而另一些像 这样的元素只能出现在某些特定元素的内部。 就是说，下面的写法会报错

<table>
  <my-row>...</my-row>
</table>

为解决上面的问题，需要使用下面的写法，关键在于is

<table>
  <tr is="my-row"></tr>
</table>

但如果使用字符串模板，则没有上面的问题，所以一般情况下，使用字符串模板吧 1. <script type="text/x-template"> 2. JavaScript 内联模板字符串 3. .vue 组件（最推荐的使用）

Vue实例中 data 必须是函数

<div id="example-2">
  <simple-counter></simple-counter>
  <simple-counter></simple-counter>
  <simple-counter></simple-counter>
</div>


var data = { counter: 0 }

Vue.component('simple-counter', {
  template: '<button v-on:click="counter += 1">{ { counter }}</button>',
  // 技术上 data 的确是一个函数了，因此 Vue 不会警告，
  // 但是我们却给每个组件实例返回了同一个对象的引用
  data: function () {
    return data//会使三个组件共享一个对象，如果不想这样，这里应该返回data的一份拷贝，Object.assign({},data)是其中一种解决方案；或者使用结构语法｛...data｝也可以，但注意，如果是深拷贝，应该注意对应事项.
  }
})

new Vue({
  el: '#example-2'
})

js中的深拷贝（知乎）

组件组合

在 Vue 中，父子组件的关系可以总结为 prop 向下传递，事件向上传递。父组件通过 prop 给子组件下发数据，子组件通过事件给父组件发送消息。看看它们是怎么工作的。

Prop

与react类似，prop只在当前组件内有效，子组件无法直接访问父组件的prop，父组件也无法直接获取子组件的prop。 子组件要显式地用 props 选项声明它预期的数据：

Vue.component('child', {
  // 声明 props
  props: ['message'],
  // 就像 data 一样，prop 也可以在模板中使用
  // 同样也可以在 vm 实例中通过 this.message 来使用
  template: '<span>{ { message }}</span>'
})

然后我们可以这样向它传入一个普通字符串：

<child message="hello!"></child>

受html解析机制，props中驼峰命名的属性需要转换为连缀才能使用

Vue.component('child', {
  // 在 JavaScript 中使用 camelCase
  props: ['myMessage'],
  template: '<span>{ { myMessage }}</span>'
})


<!-- 在 HTML 中使用 kebab-case -->
<child my-message="hello!"></child>

但这些限制在字符串模板中不会出现。

动态 Prop

与绑定到任何普通的 HTML 特性相类似，我们可以用 v-bind 来动态地将 prop 绑定到父组件的数据。每当父组件的数据变化时，该变化也会传导给子组件：

<div id="prop-example-2">
  <input v-model="parentMsg">
  <br>
  <child v-bind:my-message="parentMsg"></child>
  <!-- <child :my-message="parentMsg"></child> -->
</div>


new Vue({
  el: '#prop-example-2',
  data: {
    parentMsg: 'Message from parent'
  }
})

如果不使用:props-name，直接对对象绑定，会自动解构，

todo: {
  text: 'Learn Vue',
  isComplete: false
}

然后：

<todo-item v-bind="todo"></todo-item>

将等价于：

<todo-item
  v-bind:text="todo.text"
  v-bind:is-complete="todo.isComplete"
></todo-item>

prop的值总是字面量，如果需要其他类型，需要使用v-bind指令绑定

<!-- 传递了一个字符串 "1" -->
<comp some-prop="1"></comp>
<!-- 传递真正的数值 -->
<comp v-bind:some-prop="1"></comp>

单向数据流

与react中类似的，prop是对于子组件来说，不应该在其内部改变，改变应该放在父组件内进行， 而某些场合下，又不得不对其进行加工的话，可以使用下面两种方法修改， 1. 定义一个局部变量，并用 prop 的值初始化它：

props: ['initialCounter'],
data: function () {
  return { counter: this.initialCounter }
}

  1. 定义一个计算属性，处理 prop 的值并返回：

    props: [‘size’],
    computed: {
    normalizedSize: function () {

    return this.size.trim().toLowerCase()//对象或数组需要注意引用的问题，使用拷贝能解决大部分意外的发生

    }
    }

Prop 验证

类似于react中的PropTypes，vue也有自己的类型验证和默认值设置，

Vue.component('example', {
  props: {
    // 基础类型检测 (`null` 指允许任何类型)
    propA: Number,
    // 可能是多种类型
    propB: [String, Number],
    // 必传且是字符串
    propC: {
      type: String,
      required: true
    },
    // 数值且有默认值
    propD: {
      type: Number,
      default: 100
    },
    // 数组/对象的默认值应当由一个工厂函数返回
    propE: {
      type: Object,
      default: function () {
        return { message: 'hello' }
      }
    },
    // 自定义验证函数
    propF: {
      validator: function (value) {
        return value > 10
      }
    }
  }
})

type 可以是下面原生构造器：

String
Number
Boolean
Function
Object
Array
Symbol

type 也可以是一个自定义构造器函数，使用 instanceof 检测。 当 prop 验证失败，Vue 会抛出警告 (如果使用的是开发版本)。注意 prop 会在组件实例创建之前进行校验，所以在 default 或 validator 函数里，诸如 data、computed 或 methods 等实例属性还无法使用。这里就给人一种很怪异的感觉（有一种为了借鉴而借鉴的感觉），props和data等其他特性并不自洽。

非 Prop 特性

与react类似，你可以直接传入某些未定义的特性到组件，组件会把这些未定义的prop直接挂载到组件的根元素上。

替换/合并现有的特性

承接上文，当某些未定义的特性在组件的根元素上已经有原来的定义，外部又同时传入时，一般情况下，外部传入的值会替换组件内的值，而当特性为class或style时，两个字会进行合并。 篇幅有限，下篇再见。
