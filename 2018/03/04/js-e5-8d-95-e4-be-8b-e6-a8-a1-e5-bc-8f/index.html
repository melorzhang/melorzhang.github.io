js单例模式

博主: Melor 创建于: 2018-03-04 更新于: 2018-09-13
分类: 未分类
标签:
什么都懂一点，生活更美好一些

同样是被问起不知道该怎么回答的一个问题，调查后发现原来一般是用来开发插件类功能。具体一点就是一个类只提供一个实例，一但实例化后，就不能再次被实例化，每次实例化都会得到同一个实例的一种设计模式。之前自己还真没怎么用到这种设计模式，具体代码像下面一样。

//声明一个类
var CreateDiv = function(html) {
    this.html = html;
    this.init();
};
//定义类的方法
CreateDiv.prototype.init = function() {
    var div = document.createElement('div');
    div.innerHTML = this.html;
    document.body.appendChild(div);
}
//利用闭包，代理生成类，
var ProxySingletonCreateDiv = (function() {
    var instance;
    return function(html) {
        if (!instance) {
            instance = new CreateDiv(html);
        }
        return instance;
    }
})();

var a = new ProxySingletonCreateDiv('seven1');
//执行下面语句后，a===b结果为true，就是说b和a是同一个对象的指针，指向同一个对象（CreateDiv {html: "seven1"}）
var b = new ProxySingletonCreateDiv('seven2');

这种实现的优点在于不会重复实例化一个类，缺点也是这个类只能实例化一个实例。当需要防止重复实例化带来开销的时候，是挺有用的一个技术，可是最近好像都没这种需求
